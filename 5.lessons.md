# 经验教训

- GUI 启动阶段不能扫描大型 `weights` 目录找默认模型；应让窗口先打开，等用户开始检测时再解析默认模型。
- 判断“纯 C++ 运行端”要以 CMake 构建清单和最终 `dist` 发布包为准；源码树里的历史脚本和旧文档不等于运行端依赖，但不能让它们进入发布包。
- 发布脚本删除旧目录不能静默失败；旧 DLL 残留会误导依赖核查，也可能拖慢启动。
- 检查训练设备时，不能只看 `torch.cuda.is_available()`。它只能说明当前 Python 环境里的 PyTorch 是否支持 CUDA，不能代表电脑是否有 GPU。
- 正确顺序：先查系统硬件和 `nvidia-smi`，再查 Python/Torch 环境。
- 公开数据集验证分数高，不代表现场视频一定好。交叉带视频要单独看目标尺度、视角、背景和运动模糊。
- 合成增强只能辅助，不能替代真实交叉带人工标注；候选模型如果误检明显，不能替换默认正式权重。
- 标注工具里选中框不要使用黑色或白色，现场图片黑白背景很多，亮青色这类高对比彩色更稳。
- AI 辅助标注工具不能强依赖旧权重；旧权重不适配现场时，要优先保证手工标注、框微调和空帧清理流程顺畅。
- 标注工具加载十几万张图片时，不能用 `QListWidget.addItem()` 逐条塞 UI；要用模型列表，并把文件扫描、空标签检查放到后台线程。
- mAP 高不等于现场效果好；如果验证集里有同样的错误大框，模型学会框住交叉带平台也会拿到高分。包裹数据必须保留真实空帧负样本，并清理面积异常的大框。
- 标注工具判断 SAM 是否可用时，不能只检查 `ultralytics` 包是否存在；必须同时检查 `torch` 和 `SAM` 类能否真实导入，否则基础发布包会误显示 SAM 已加载。
- Windows 打包脚本不能静默忽略旧发布目录删除失败；要先检查旧 exe 是否仍在运行，并且每个外部命令都要检查退出码。
- 打 AI 发布包时不能用 `--extra-index-url` 让 pip 自己选 Torch；它可能选到 CPU 版。CUDA 版必须先用 `--index-url https://download.pytorch.org/whl/cu128` 安装 Torch/TorchVision。
- 桌面检测工具左侧参数栏不能只用固定宽度 QWidget；要用可滚动容器和 splitter stretch，窗口最大化或分辨率变化时才不会挤在一起。
- 运行耗时展示要和检测指标单位一致；DWS 工具的 read/infer/total 都是 ms，GUI 当前耗时也必须显示 ms。
- PyInstaller one-folder 包的数据资源默认在 `_internal`，程序里找默认配置、模型、文档时要优先使用 `sys._MEIPASS`，不能只按 exe 同级目录找。
- Ultralytics OpenVINO 推理需要选择 `*_openvino_model` 目录或其中的 `.xml`；同目录必须有对应 `.bin`，打包时还要把 `openvino` 依赖和 DLL 一起收进去。
- Windows Application Control / Device Guard 拦截发布 exe 时，不能只在脚本里忽略警告；要先确认是否是未签名导致，再在发布脚本里签名 build exe、dist exe 和安装包，并让烟测真实失败或通过。
- OpenCode 本地大模型优化不能为了速度降低模型能力；上下文应按服务端真实上限配置，优先解决扫描目录、代理绕路、入口错误和缓存问题。
- Windows 失败产物和修复产物不能使用相同版本号并同时保留；修复后必须升版本、在窗口显示版本，并只交付路径明确的最终安装包，避免误开旧包。
- 给普通用户交付 PowerShell 工具时，不能把 `.ps1` 作为可见主入口；必须提供名称明确的 `.bat` 双击入口，并隐藏后台脚本，避免被编辑器打开。
- Windows 盘符路径不能直接交给通用 URL 解析后判断类型；只有字符串明确包含 `://` 时才按网络地址处理，否则 `D:\...` 会被误显示成 `D.网络视频流`。
- “无流量持续时间”和“堵包持续时间”是两个指标；界面显示堵包秒数时必须先确认堵包状态，未堵包必须显示 0。
- 视频流的“应用”动作必须立即建立独立预览，不能把看到画面和开始推理解耦错误地绑在一起；预览读取必须放在线程中，避免相机异常卡住界面。
- 现场 ROI 如果定义为本次会话配置，就不能从 `QSettings` 或默认区域文件自动恢复；软件重启必须从空 ROI 开始。
- RTSP 通道切换时，界面线程不能调用 `QThread::wait()` 等待采集线程退出；应先发停止信号，只保留最后一次切换请求，旧线程结束后再异步连接新通道。仅软件退出时允许等待线程收尾。
- OpenVINO C++ 运行包不能只复制 `openvino.dll` 和 device plugin；读取 `.xml` 还必须带 `openvino_ir_frontend.dll`。打包脚本排除 debug DLL 时不能用泛泛的 `d.dll` 后缀判断，否则会误删 `frontend.dll`。
- YOLO 分割 OpenVINO 端到端输出如果是 `[1,N,6+mask]`，前 6 列才是框、置信度和类别；后面的 mask 系数不能当类别分数，否则空画面也会出现大量错误大框。
- OpenCV `cv::putText` 自带 Hershey 字体不支持中文；视频帧叠加文字如果直接传中文会变成问号，应使用 ASCII 标识或改用 Qt/FreeType 绘制中文。
- C++ ByteTrack 不能只按检测置信度过滤后再跟踪；要保留低分候选用于续跟，并把新轨迹阈值与 Ultralytics `new_track_thresh` 对齐，否则比 Python tracker 更容易断 ID。
- 流量监测可视化要按目标框中心点是否在 ROI 内变色：ROI 外黄色，中心点进入或压线后绿色，中心点离开后恢复黄色，不能只用固定检测框颜色。
- 流量计数本身可以每帧计算，但如果看板 payload 只跟随预览帧发送，用户看到的累计数量仍会延迟；统计 payload 应每帧发，图像预览可以降频发。
