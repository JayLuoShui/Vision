# AGENTS.md | Vision 项目 OpenCode 规则

## 启动规则
- 先读 `CONTEXT.md` 和 `5.lessons.md`，再决定是否读 `README.md`、`ARCHITECTURE.md`。
- 需求明确时直接做；遇到删除文件、数据库写入、创建分支、创建 worktree、推送、重置代码这类高风险动作，必须先问。
- 不为了速度降低主模型能力，不擅自缩小上下文上限。

## 工作方式
- 用简洁中文回复，先说结论，再说必要过程。
- 简单任务直接改；复杂任务先给短计划。
- 只改完成任务必须改的文件，不做无关重构。
- 改完先自查 Bug，再从原始需求复盘是否更简单。
- 能跑验证就跑，最终只报告通过或失败数量。

## 项目记录
- 阶段性完成后必须更新 `CONTEXT.md`。
- 只有新增或删除功能模块、运行/部署方式变化、重要设计决定变化时，才更新 `README.md` 或 `ARCHITECTURE.md`。
- 用户指出规则或做法不合适时，立刻把教训写入 `5.lessons.md`。

## 在线搜索
- 涉及最新版本、官方文档、MCP、模型、依赖、法规或价格时，必须联网核对。
- 优先使用当前项目启用的免费搜索 MCP；搜索结论要保留来源链接。
- 不把 API Key 写入仓库。

## OpenCode 自我迭代
- 当前规则不是一次性写死的配置；如果使用中发现啰嗦、乱改、不问高风险动作、不会搜索，就更新 `AGENTS.md`、`.opencode/agents/`、`.opencode/commands/` 或 `OPENCODE_TUNING.md`。
