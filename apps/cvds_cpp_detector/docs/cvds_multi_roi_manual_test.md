# CVDS 多 ROI 看板手工验收

## 区域配置

- [ ] 新增 3 个区域并分别命名为主线入口、左分流口、右分流口。
- [ ] 每次切换选中区域后，绘制、撤回和清空只影响当前区域。
- [ ] 区域名称、计数开关、堵包开关和堵包秒数可以修改。
- [ ] 删除一个区域后，画面和区域表同时移除该区域。
- [ ] 保存配置后关闭软件，重新启动并加载，区域和主统计区域保持一致。
- [ ] 配置缺少区域、区域 ID 重复或多边形少于 3 点时，软件明确报错。

## 检测与统计

- [ ] worker 启动参数包含 `--regions`，输出目录生成 `regions.json`。
- [ ] 包裹依次经过主线入口和分流口时，各区域独立计数。
- [ ] 顶部累计数量只等于主统计区域数量，不等于全部区域数量之和。
- [ ] 多个 ROI 重叠时，同一目标可进入多个区域，但顶部总数仍只取主统计区域。
- [ ] `flow_events.csv` 包含区域 ID、区域名称和区域累计数量。
- [ ] `flow_summary.json` 包含 `total_count_region` 和全部区域汇总。

## 堵包与看板

- [ ] 区域内有包裹且数量连续达到设定秒数不变化时，该区域进入堵包。
- [ ] 堵包时顶部状态、视频边框和对应区域行按约 500ms 红色闪烁。
- [ ] `jam_signals.jsonl` 记录区域 ID、区域名称和 `IO_JAM_ON`。
- [ ] 包裹离开或区域数量变化后，堵包解除，界面立即恢复。
- [ ] 解除事件记录 `IO_JAM_OFF`。
- [ ] 一个区域解除但另一个区域仍堵包时，全局状态继续显示堵包。

## 兼容与发布

- [ ] 旧 `--roi` 命令仍能运行并自动使用 `default` 区域。
- [ ] 发布目录包含 `configs/regions.example.json`。
- [ ] 无 Python、Qt、OpenCV 的目标电脑可启动 GUI 和 worker。
