# 部署说明

## 构建安装包

开发机需要 Visual Studio Build Tools、CMake、Ninja、Qt 6、OpenCV 和 Inno Setup。

```powershell
powershell -ExecutionPolicy Bypass -File .\apps\cvds_cpp_detector\packaging\build_release.ps1
```

脚本会完成：

1. 编译 C++/Qt 主程序。
2. 使用 `windeployqt` 收集 Qt DLL。
3. 收集 OpenCV DLL。
4. 创建隔离 venv，安装 worker 依赖。
5. 使用 PyInstaller 生成 `runtime/cvds_detector_worker.exe`。
6. 复制 `weights`、`configs`、`docs`、版本文件。
7. 执行 worker 自检和模型读取 smoke test。
8. 调用 Inno Setup 生成安装包。

## 安装后目录

- 程序目录：`{autopf}\CVDS\CVDS包裹流量检测工具`
- worker：`runtime/cvds_detector_worker.exe`
- tracker：`configs/bytetrack.yaml`
- 多 ROI 示例：`configs/regions.example.json`
- 权重：`weights/`
- 文档：`apps/cvds_cpp_detector/docs/`

## 运行数据目录

检测输出写入用户目录，不写入安装目录：

```text
%LOCALAPPDATA%/CVDS/CVDS包裹流量检测工具/runs/
```

长期区域配置写入：

```text
%LOCALAPPDATA%/CVDS/CVDS包裹流量检测工具/configs/regions.json
```

## GPU 说明

CPU 模式是基础可运行形态。GPU 是可选加速，需要电脑有 NVIDIA GPU、匹配驱动，以及发布包内 PyTorch 支持 CUDA。
