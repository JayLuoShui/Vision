# 部署说明

当前部署对象是 `CVDS_Cpp_Detector.exe`，不是 Python worker 包，也不是独立 WCS 程序。

## 发布产物

默认命令：

```powershell
pwsh -ExecutionPolicy Bypass -File .\apps\cvds_cpp_detector\packaging\build_release.ps1
```

默认输出：

```text
dist\CVDS_Cpp_Detector\CVDS_Cpp_Detector.exe
dist_installer\CVDS_Cpp_Detector_Setup_<version>.exe
```

避免覆盖旧发布包：

```powershell
pwsh -ExecutionPolicy Bypass -File .\apps\cvds_cpp_detector\packaging\build_release.ps1 `
  -DistName CVDS_Cpp_Detector_TestBuild
```

## 部署内容

便携目录包含：

- `CVDS_Cpp_Detector.exe`
- Qt / OpenCV / OpenVINO / TBB / MSVC 运行库
- 可选 TensorRT Runtime DLL
- `models/` 下的 OpenVINO IR `.xml + .bin`
- `VERSION.txt`
- `README_RELEASE.md`

不包含：

- Python
- conda
- Torch
- Ultralytics
- PyInstaller worker
- `.pt` 或 `.onnx` 模型

## 现场运行

1. 打开 `CVDS_Cpp_Detector.exe`。
2. 选择本地视频或填写海康 RTSP 参数。
3. 点击应用视频源，确认预览画面正常。
4. 绘制 ROI。
5. 选择 OpenVINO 或 TensorRT 后端。
6. 点击开始检测。

## 数据输出

输出目录由界面选择。多路检测时，每路写到 `camera_1`、`camera_2` 等子目录。

每路目录包含：

- `cvds_online_parcel_flow_monitor.mp4`
- `flow_events.csv`
- `jam_signals.jsonl`
- `flow_summary.json`
- `cvds_preview.jpg`

## 设备说明

- OpenVINO 界面设备：`AUTO`、`CPU`、`GPU`。
- TensorRT 设备：NVIDIA CUDA GPU 编号。
- 显式选择不可用设备时应直接报错，不做静默切换。
