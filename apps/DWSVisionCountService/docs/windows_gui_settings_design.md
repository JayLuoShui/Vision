# Windows GUI 设置与 ROI 设计

## 目标

- 在一个窗口内完成服务监控、运行参数设置和 ROI 标定。
- 配置保存到 `config/config.yaml`，保存成功后完整重启检测服务。
- ROI 坐标始终使用原始图片坐标，不受窗口缩放影响。
- 使用用户指定的蓝色图案作为窗口、EXE 和安装包图标。

## 界面

- 左侧：品牌图标、服务状态、监听地址、请求数、最近结果和启动/停止按钮。
- 右侧“运行设置”：TCP 端口、模型目录、置信度、IoU、推理线程、JPEG reduce 倍率和调试图开关。
- 右侧“ROI 标定”：选择现场图片、绘制检测矩形、输送带多边形和多个忽略矩形，支持撤销和清空。

## 生效规则

- 保存前严格检查端口、阈值、线程数、reduce 倍率、模型路径和 ROI。
- 不修改正在运行的配置对象。
- 先写入 YAML，再停止旧服务，最后使用新配置创建全新服务。
- 保存或重启失败时明确报错，不做降级或静默回退。

## ROI 操作

- 检测区域：拖动绘制一个矩形。
- 输送带区域：逐点单击，双击或点击“完成多边形”闭合，至少三个点。
- 忽略区域：每次拖动新增一个矩形。
- 撤销只撤回当前模式最近一次操作；清空只清空当前模式。

## 验证

- 配置 YAML 往返保存测试。
- 参数范围和 ROI 合法性测试。
- 画布与原图坐标双向映射测试。
- 服务使用新配置完整重启测试。
- 打包后启动、签名、TCP JPEG bytes 请求和窗口图标检查。
