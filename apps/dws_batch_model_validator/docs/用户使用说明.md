# 用户使用说明

## 软件安装

运行 `DWSBatchModelValidator_Setup_1.0.0.exe`，按提示安装。也可以直接解压 `DWSBatchModelValidator.zip` 后双击 `DWSBatchModelValidator.exe`。

## 第一次启动

双击 exe 会打开图形界面。程序默认把输出写入 `%LOCALAPPDATA%/CVDS/DWSBatchModelValidator/outputs/runs`。

## 选择模型

在“模型文件”中可以选择 `.pt` YOLO segmentation 权重，也可以选择 Ultralytics 导出的 OpenVINO `.xml` 文件。OpenVINO 导出目录通常叫 `xxx_openvino_model`，目录里会同时有 `.xml` 和 `.bin`。模型不会强制内置到安装包。

## 选择数据

选择图片目录和 YOLO segmentation 标签目录。每张图片会查找同名 `.txt` 标签，缺失标签会继续推理，但不参与有标签指标。

## 设置阈值

`low_conf` 用于可疑候选，`high_conf` 用于确认包裹。默认 `low_conf=0.25`，`high_conf=0.55`，`iou=0.50`。

## 选择 CPU/GPU

“自动”会优先使用 CUDA GPU，没有 CUDA 就使用 CPU。“GPU”会强制检查 CUDA，不可用时给出中文错误。OpenVINO 模型固定按 OpenVINO CPU 推理运行。

## 开始检测

点击“开始检测”。运行时会显示进度、当前图片、状态、信号、日志和最近一张可视化图片。点击“停止/取消”会保留已完成结果。

## 查看结果

输出目录中包含：

- `results.csv`
- `summary.json`
- `resolved_config.json`
- `vis/`
- `errors/false_single`
- `errors/false_multi`
- `errors/unknown`

## 状态含义

- `SINGLE`：确认只有一个包裹。
- `MULTI`：确认有多个包裹。
- `SUSPECT_MULTI`：一个确认包裹加一个低置信候选。
- `UNKNOWN`：没有确认包裹或图片处理失败。

## 信号含义

- `PASS`：放行。
- `INTERCEPT`：拦截。
- `INTERCEPT_REVIEW`：拦截并复核。
- `REVIEW`：人工复核。
