# 故障排查

## 程序打不开

先运行 `DWSBatchModelValidator.exe --diagnose` 查看环境信息。

## 找不到模型

重新选择 `.pt` 文件，或选择 OpenVINO 导出目录里的 `.xml` 文件。程序不会默认把大模型强制打进安装包。

## OpenVINO 模型不能运行

确认选择的是 Ultralytics 导出的 `*_openvino_model` 目录里的 `.xml` 文件，并且同目录下有同名 `.bin` 文件。源码运行时需要安装 `openvino>=2024.0.0`。

## 图片目录为空

确认目录下有 `.jpg/.jpeg/.png/.bmp` 图片。

## torch 或 ultralytics 导入失败

说明当前发布包或源码环境缺少 AI 推理依赖。重新安装 requirements 或重新打包。

## CUDA 不可用

改用 CPU，或安装匹配的 NVIDIA 驱动和 PyTorch CUDA 运行环境。

## GPU 模式失败

强制 GPU 时必须检测到 CUDA。自动模式可回退 CPU。

## 中文路径读取失败

当前版本已使用 `np.fromfile + cv2.imdecode` 和 `cv2.imencode + write_bytes` 处理中文路径；仍失败时检查文件权限和图片是否损坏。

## results.csv 没有生成

检查输出目录是否可写。默认输出在用户 AppData，不写 Program Files。

## 可视化图片没有生成

确认勾选“保存可视化图片”。如果只想看错误图，也要打开“保存错误样本”。

## 检测速度慢

CPU 会慢于 GPU。可以减小输入尺寸，或在 CUDA 可用时选择自动/GPU。

## false_single 较多

适当降低 `high_conf` 或降低 `low_conf`，让疑似第二件包裹进入 `SUSPECT_MULTI`。

## false_multi 较多

适当提高 `high_conf` 或提高 `iou`，减少重复框和低质量候选。
